# BnB Plugin Implementation Plan (Plan 2 of 4)

> **For agentic workers:** Steps use checkbox (`- [ ]`) syntax for tracking. Implement task-by-task, TDD where there is testable structure; commit per task.

**Goal:** Build the Claude Code plugin: manifest + marketplace packaging, the 8 phase commands (`/bnb` status router + idea/prd/architecture/scaffold/feature/overnight/doctor), the 5 self-contained skills (bnb-prd, bnb-architecture, bnb-feature, bnb-debugging, bnb-overnight), the artifact templates (§5), and the Stop-hook wiring — all validated by structural tests in `node:test`.

**Architecture:** Everything lives under `plugin/` (spec §3) so the whole directory is installable as one plugin. Commands are thin markdown routers that set phase context and hand off to skills; skills carry the methodology (hard rules up top, red-flag tables per spec §12); templates are the §5 artifact skeletons the scaffold/prd phases copy into target projects. A structural test suite (`test/plugin.test.js`) keeps the tree honest: manifest valid, every command routes to a real skill, every skill has frontmatter, hook script exists and behaves identically to `bin/bnb-stop-hook.js`.

**Tech Stack:** Markdown + JSON; one standalone Node script (`plugin/hooks/verify-before-stop.js`); tests in `node:test`. Still zero dependencies, no build step.

## Resolved design decisions (spec §15 Q1)

Verified against current Claude Code plugin docs (code.claude.com/docs/en/plugins-reference, checked 2026-07-08):

- **Manifest**: `plugin/.claude-plugin/plugin.json`; only `name` is required. Plugin **name = `bnb`** — the name namespaces components (`bnb:idea` etc.), giving the spec's `/bnb:*` prefix for free. npm package stays `breathe-and-build` (§2 naming decision: search collision only matters for the published package/repo).
- **Commands**: flat `.md` files in `plugin/commands/` (NOT `commands/bnb/` as sketched in §3 — the plugin name already provides the namespace; a subdir would double it to `/bnb:bnb:idea`). Status router = `plugin/commands/bnb.md` → `/bnb`.
- **Skills**: `plugin/skills/<name>/SKILL.md` with `name` + `description` frontmatter.
- **Hooks**: `plugin/hooks/hooks.json`, `Stop` event, command `node "${CLAUDE_PLUGIN_ROOT}/hooks/verify-before-stop.js"`. `${CLAUDE_PLUGIN_ROOT}` resolves to `plugin/` only — a marketplace install copies just the plugin dir, so the hook script CANNOT `import` from `../src/`. It is a self-contained duplicate of the stop-hook logic, kept honest by a parity test.
- **Marketplace**: repo-root `.claude-plugin/marketplace.json` naming marketplace `breathe-and-build`, one plugin entry `{ "name": "bnb", "source": "./plugin" }`. Local install for testing: `claude plugin marketplace add <repo-path>` then `claude plugin install bnb@breathe-and-build`. The npx installer (Plan 4) automates this.

## Global Constraints

- Do not touch `src/core/*` or `bin/*` contracts (Plan 1, frozen — see README).
- Skills are self-contained: no references to superpowers or any external plugin.
- Skills follow writing-skills discipline: hard rules first, a red-flags table, imperative voice.
- Every command file: frontmatter with `description` (+ `argument-hint` where args exist), body that sets phase context and routes into a skill or CLI. Commands stay under ~40 lines; methodology lives in skills.
- Profile-specific content (Expo/Maestro specifics) belongs to Plan 3; skills reference "the profile's toolbelt/conventions docs" generically and `bnb.config.json` fields, never hardcoded stack commands. (The bnb-feature skill may cite `maestro` in examples marked as profile-example.)
- Templates use `{{PLACEHOLDER}}` markers for scaffold-time substitution.
- Tests: extend `npm test`; structural checks live in `test/plugin.test.js`, hook parity in `test/plugin-hook.test.js`.

### Task 2: Stop-hook wiring (self-contained)

**Files:**
- Create: `plugin/hooks/hooks.json`, `plugin/hooks/verify-before-stop.js`, `test/plugin-hook.test.js`

`hooks.json`:

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "node \"${CLAUDE_PLUGIN_ROOT}/hooks/verify-before-stop.js\""
          }
        ]
      }
    ]
  }
}
```

`verify-before-stop.js` — standalone (no imports from `src/`): read stdin JSON (`cwd` fallback `process.cwd()`); no `progress.json` → exit 0; unreadable → exit 0 (never wedge); tasks with `active && !passes && !abandoned` → print the same reason text as `src/hooks/stop-decision.js` to stderr, exit 2; else exit 0.

`test/plugin-hook.test.js` — spawn-based **parity suite**: run BOTH `bin/bnb-stop-hook.js` and `plugin/hooks/verify-before-stop.js` against the same fixtures (no file / red task / green / abandoned / invalid JSON) and assert identical exit codes; assert red-task stderr names the task and mentions `abandoned`. Also assert `hooks.json` references `${CLAUDE_PLUGIN_ROOT}/hooks/verify-before-stop.js` and the file exists next to it.

- [ ] Steps: tests (red) → implement → green → commit `feat(plugin): Stop hook wiring — self-contained verify-before-stop with parity test vs bnb-stop-hook`.

### Task 3: Artifact templates (§5)

**Files:**
- Create under `plugin/templates/`: `prd.md`, `state.md`, `debug.md`, `progress.json`, `spiritual-guide.md`, `remaining-tasks.md`, `CLAUDE.md`

Requirements per template:

- **prd.md** — vision/users/roles sections; screen-spec format: `S-NN <Name>` blocks with **Purpose / Elements / Behaviour / Design notes / testIDs[]** and `R-NN` role-variant sub-blocks; data lifecycle + permissions matrix sections; a `Gaps log` section (the /bnb:prd loop appends `gaps found: N` per pass); one fully worked example screen block marked as example.
- **state.md** — stack, status-by-area table, conventions, gotchas, "read this first" preamble; updated-at-every-checkpoint instruction.
- **debug.md** — numbered ledger format: `## Bug N — <symptom>` with Symptom/Root cause/Fix/Lesson; instruction to cite entries ("see Bug 8").
- **progress.json** — `{ "tasks": [] }` plus a `_schema` comment-task showing all fields (id/spec/acceptance/testIDs/flow/active/passes/abandoned/reflexions) matching README format.
- **spiritual-guide.md** — resolves spec §15 Q3: prompts that elicit business essence — who is this for; the one job the app must never fail at; what to choose when speed vs correctness conflict; what the user would cut first; tone/brand notes; "when in doubt" priority list. Header states its role: answer-of-last-resort for unattended runs.
- **remaining-tasks.md** — cross-session skip ledger: table of deferred item / why deferred / trigger to revisit.
- **CLAUDE.md** — per-project memory: read state.md + debug.md first; progress.json is the resume artifact; run `bnb-verify <task>` for the green gate, never edit `passes`; commit only after verified; consult profile toolbelt/conventions docs; `{{PROFILE}}`/`{{PROJECT_NAME}}` placeholders.

Tests (extend `test/plugin.test.js`): all 7 files exist; prd.md contains `testIDs` and an `S-01` example; progress.json parses with `tasks: []`; CLAUDE.md mentions `bnb-verify` and forbids editing `passes`; spiritual-guide.md contains at least 4 `##` prompt sections.

- [ ] Steps: tests (red) → write templates → green → commit `feat(plugin): artifact templates — prd screen-spec, state, debug, progress, spiritual-guide, CLAUDE.md`.

### Task 4: Commands (8 thin routers)

**Files:**
- Create under `plugin/commands/`: `bnb.md`, `idea.md`, `prd.md`, `architecture.md`, `scaffold.md`, `feature.md`, `overnight.md`, `doctor.md`

Contracts (each ≤ ~40 lines, frontmatter `description` [+ `argument-hint`]):

- **bnb.md** (status router): read `state.md` + `progress.json` if present; determine phase (no prd.md → idea; prd.md not final → prd; no architecture decision in state.md → architecture; no scaffold → scaffold; else feature/overnight); output "You are in <phase>. Next: /bnb:<command>". Never starts work itself.
- **idea.md**: long free-form conversation; ends by drafting `prd.md` v0 from `plugin/templates/prd.md`; human gate = user confirms direction; then point at `/bnb:prd`.
- **prd.md**: route into skill `bnb-prd`.
- **architecture.md**: route into skill `bnb-architecture`; includes the unsupported-stack honesty rule (§7): no profile → offer supported profile or best-effort custom verifier.json, user owns quality.
- **scaffold.md**: prerequisites via `bnb-doctor` first (exit 3 → stop with fix-it list); birth project from bound profile templates (Plan 3; until it lands, copy `plugin/templates/*` artifacts and state honestly that stack scaffolding requires the profile); user must fill `spiritual-guide.md`; human gate = scaffold review.
- **feature.md** (`argument-hint: <task-id>`): route into skill `bnb-feature`.
- **overnight.md** (`argument-hint: <task-ids...>`): route into skill `bnb-overnight`.
- **doctor.md**: run `bnb-doctor` CLI, relay fix-it list; explain exit 0/3.

Tests (extend `test/plugin.test.js`): exactly these 8 files; each has frontmatter with non-empty `description`; `feature.md`/`overnight.md` have `argument-hint`; each of prd/architecture/feature/overnight names its skill and that skill dir exists (generic assertion: command body mentioning `skills/<x>` or skill name `bnb-<x>` must correspond to an existing skill).

- [ ] Steps: tests (red) → write commands → green → commit `feat(plugin): 8 phase commands — /bnb status router + thin skill routers`.

### Task 5: Planning-phase skills — bnb-prd, bnb-architecture

**Files:**
- Create: `plugin/skills/bnb-prd/SKILL.md`, `plugin/skills/bnb-architecture/SKILL.md`

**bnb-prd** (spec §4 gap-hunting loop): HARD RULES: read the ENTIRE prd.md each pass; ask about gaps ONE AT A TIME; every screen gets an S-NN block with testIDs[] (E2E-ready by construction); end every pass with `gaps found: N`; exit only when N=0 AND user says final. Gap taxonomy: dead-end journeys, unspecified states (empty/loading/error/offline), role/permission holes, data lifecycle gaps (create/edit/delete/archive), cross-screen consistency. Red-flags table: "asked 5 questions at once", "declared final after 1 pass", "screen without testIDs", "user said 'looks good' ≠ said 'final'".

**bnb-architecture** (spec §4): facilitate review of DB schema, API surface, offline/sync strategy — present for the user to challenge; confirm stack (default expo-react-native profile; overriding → warn about losing tuned verifier, §7 honesty rule); output an architecture section into state.md; human gate = explicit approval. Red-flags: "designed without reading prd.md", "picked stack silently", "skipped offline/sync question for a mobile app".

Tests: SKILL.md frontmatter `name` matches dir, non-empty `description`; bnb-prd contains `gaps found:` and a red-flags section; bnb-architecture mentions the default profile + warn-on-override.

- [ ] Steps: tests (red) → write skills → green → commit `feat(plugin): planning skills — bnb-prd gap-hunting loop, bnb-architecture review`.

### Task 6: Build-phase skills — bnb-feature, bnb-debugging

**Files:**
- Create: `plugin/skills/bnb-feature/SKILL.md`, `plugin/skills/bnb-debugging/SKILL.md`

**bnb-feature** — the green loop, spec §8 steps 1–6 verbatim as the skill's process (Plan → Implement (see it render; blank/black screenshot = crash not done) → Author E2E first + commit flow before chasing green → `bnb-verify <id>` → Self-correct via subagent (spec: subagent gets spec/acceptance/testIDs + failing tail + prior reflexions[]; returns hypothesis + evidence + suggested fix ≤250 words; NEVER edits files; main loop appends Reflexion, applies fix) → Checkpoint: code review as tamper check, commit, `active:false`, update state.md/debug.md). Anti-reward-hacking red-flags table from §9 verbatim: "unit passes so it's done", "I'll tweak the flow", "context is long, let me wrap up" → reset + resume from progress.json. HARD RULES: never edit `passes`; never weaken verifier/flows; exit code + failing tail are the only evidence of green.

**bnb-debugging** — evidence-before-hypothesis; investigation DELEGATED to a subagent returning capped summaries (token economy §10); read debug.md FIRST and cite entries ("see Bug 8"); append a new numbered entry after every root-cause fix; Reflexion entries in progress.json for in-loop fixes. Red-flags: "fixed it without knowing why it broke", "same symptom as Bug N but didn't read Bug N", "pasted 300 lines of logs into context".

Tests: frontmatter as Task 5; bnb-feature mentions `bnb-verify`, `reward hacking`, commit-flow-before-green, and a subagent-investigation rule; bnb-debugging mentions `debug.md` citation and subagent delegation.

- [ ] Steps: tests (red) → write → green → commit `feat(plugin): build skills — bnb-feature green loop, bnb-debugging evidence-first`.
